## ADDED Requirements

### Requirement: 手动两栏标注工作台
系统 MUST 为 `manual` 计划展示手动标注工作台，并 SHALL 使用两栏布局展示病历原文和手动标注条目。

#### Scenario: 手动任务成功加载
- **GIVEN** 下一条任务 API 返回 `annotation_type=manual` 的任务 payload
- **WHEN** 标注页面渲染
- **THEN** 页面展示两栏结构，左侧为病历原文，右侧为手动标注条目面板
- **AND** 左侧病历原文栏 title 区展示住院号和“查看质控规则”入口
- **AND** 首次进入且尚未创建条目时，右侧展示空态
- **AND** 页面不展示输出 A 或输出 B 面板

#### Scenario: 手动任务存在长病历内容
- **GIVEN** 手动任务的病历文本超过视口高度
- **WHEN** 标注页面渲染
- **THEN** 病历原文区域支持独立滚动
- **AND** 右侧标注条目面板和提交操作仍然可访问

### Requirement: 手动标注条目创建、编辑与删除
系统 SHALL 支持操作员通过选择病历原文快速创建手动标注条目，并 MUST 在当前病例提交前支持标注条目卡片的查看、新增、编辑和删除。系统 MUST 在提交前校验每条问题条目的必填字段。系统 MUST NOT 提供右侧面板固定“新增条目”入口，新增只能从左侧病历原文划选触发。

#### Scenario: 划选病历原文创建条目
- **GIVEN** 操作员正在查看手动任务
- **WHEN** 操作员在左侧病历原文中选择一段文本
- **THEN** 页面在划线结束位置旁边展示“标注”浮层按钮
- **AND** 浮层按钮不得固定在病历原文区域的某个静态位置
- **WHEN** 操作员再次点击病历原文并取消当前划选
- **THEN** “标注”浮层按钮随当前划选一起消失
- **WHEN** 操作员点击“标注”
- **THEN** 右侧创建一条标注条目
- **AND** 新建条目表单的上边沿在无视觉遮挡时与所选原文片段上边沿保持同一水平位置
- **AND** 右侧条目面板自动滚动到新建表单所在位置，不要求操作员手动寻找该表单
- **AND** 条目的 `source_text` 自动填充为选中的病历原文片段
- **AND** 条目的 `start_offset` 和 `end_offset` 自动填充为该片段在清洗后 `record_text` 中的字符范围
- **AND** `start_offset` 和 `end_offset` 不作为表单字段展示，也不允许标注员编辑

#### Scenario: 跨行原文片段被保留
- **GIVEN** 操作员选择的病历原文片段包含换行
- **WHEN** 系统创建手动标注条目
- **THEN** 条目的 `source_text` 保留换行信息，并可在提交时规范为 `\n`
- **AND** 条目的 `start_offset` 和 `end_offset` 覆盖完整跨行片段

#### Scenario: 填写问题条目
- **GIVEN** 操作员已经创建一条手动标注条目
- **WHEN** 操作员查看只读原文片段、选择一个质控规则并填写修改建议
- **THEN** 该条目满足提交前字段要求
- **AND** 质控规则选择控件按规则类型分组展示规则，支持按规则内容模糊搜索
- **AND** 备注字段保持选填
- **AND** 条目表单不展示原文定位数值

#### Scenario: 编辑未提交条目
- **GIVEN** 操作员正在查看尚未提交的手动病例
- **AND** 右侧已经存在一条标注条目卡片
- **WHEN** 操作员点击条目卡片右上角编辑图标
- **THEN** 该条目进入编辑态，编辑表单在原条目卡片位置替换展示
- **AND** 表单展示原文片段只读、按规则类型分组且支持搜索的质控规则选择、修改建议和备注
- **WHEN** 操作员修改该条目的质控规则、修改建议或备注并保存
- **THEN** 页面更新该条目卡片内容
- **AND** 页面展示保存成功 toast 反馈
- **AND** 修改后的内容参与最终提交

#### Scenario: 删除未提交条目
- **GIVEN** 操作员正在查看尚未提交的手动病例
- **AND** 右侧已经存在一条标注条目卡片
- **WHEN** 操作员点击条目卡片右上角删除图标
- **THEN** 页面使用系统反馈组件的 Dialog 样式展示二次确认弹窗
- **WHEN** 操作员取消删除
- **THEN** 页面关闭弹窗并保留该条目
- **WHEN** 操作员确认删除
- **THEN** 页面从右侧条目面板移除该条目
- **AND** 左侧对应病历原文片段不再高亮
- **AND** 被删除条目不参与最终提交

#### Scenario: 问题条目缺少规则或修改建议
- **GIVEN** 操作员已经创建一条手动标注条目
- **WHEN** 条目缺少质控规则或修改建议时保存
- **THEN** 客户端阻止保存，并在对应表单项下展示字段级校验提示

### Requirement: 手动条目高亮与定位
系统 MUST 在手动工作台中将每个标注条目卡片与其引用的病历原文片段建立可视关联。左侧被引用的病历原文片段 MUST 默认使用黄色高亮展示，右侧条目卡片 SHALL 基于 `start_offset` / `end_offset` 尽量与对应高亮片段保持同一水平位置。左右两栏 SHALL 共享同一纵向画布语义：任一侧滚动或定位到某个 offset 时，另一侧应同步滚动到对应上下文。

#### Scenario: 创建条目后高亮原文
- **GIVEN** 操作员正在查看手动任务
- **WHEN** 操作员划选病历原文并创建标注条目
- **THEN** 左侧被引用的病历原文片段以黄色高亮展示
- **AND** 右侧出现对应标注条目卡片
- **AND** 高亮范围由该条目的 `start_offset` 和 `end_offset` 确定

#### Scenario: 条目卡片与原文片段对齐
- **GIVEN** 手动工作台中存在一个标注条目
- **WHEN** 页面展示该条目卡片
- **THEN** 在没有视觉遮挡的前提下，条目卡片上边沿在右侧面板中的纵向位置与左侧对应高亮原文片段上边沿保持同一水平位置

#### Scenario: 左右两栏滚动联动
- **GIVEN** 手动工作台中的病历原文较长
- **WHEN** 操作员滚动左侧病历原文区域
- **THEN** 右侧标注条目面板按照同一纵向画布坐标同步滚动
- **WHEN** 操作员滚动右侧标注条目面板
- **THEN** 左侧病历原文区域同步滚动到对应上下文

#### Scenario: 选中条目后突出对应原文
- **GIVEN** 手动工作台中存在多个标注条目
- **WHEN** 操作员点击右侧某一个标注条目卡片
- **THEN** 左侧该条目对应的病历原文片段从默认黄色高亮切换为浅红色高亮
- **AND** 左侧病历原文区域自动滚动到该条目对应原文片段附近，便于查看上下文
- **AND** 其他未选中条目对应原文片段保持黄色高亮
- **AND** 被选中条目卡片进入稳定选中态，鼠标移出后仍保持选中视觉
- **AND** 鼠标点击其他空白区域后，条目退出选中态并恢复默认灰色边框
- **AND** 条目卡片默认态为灰色边框，hover 态为蓝色边框，选中态为稳定的蓝色边框和浅蓝底色

#### Scenario: 功能图标状态
- **GIVEN** 手动工作台中存在未提交的标注条目
- **WHEN** 操作员 hover 或 click 编辑图标
- **THEN** 编辑图标展示可感知的 hover 和 click 视觉状态
- **WHEN** 操作员 hover 或 click 删除图标
- **THEN** 删除图标展示可感知的危险操作 hover 和 click 视觉状态

#### Scenario: 根据持久化定位恢复高亮
- **GIVEN** 手动工作台加载已有标注条目数据
- **AND** 条目包含 `source_text`、`start_offset` 和 `end_offset`
- **WHEN** 页面渲染病历原文和条目卡片
- **THEN** 页面根据 `start_offset` 和 `end_offset` 恢复左侧黄色高亮
- **AND** 右侧条目卡片与恢复后的高亮片段纵向对齐

#### Scenario: 多条卡片防止重叠
- **GIVEN** 多个高亮原文片段在左侧距离过近
- **WHEN** 右侧对应条目卡片按原始位置展示会产生重叠
- **THEN** 右侧条目卡片以非重叠方式平铺展示完整卡片内容
- **AND** 引用原文位置更靠前的卡片排在更上方

#### Scenario: 删除条目后重新布局
- **GIVEN** 右侧存在多个标注条目卡片
- **WHEN** 操作员删除其中一个未提交条目
- **THEN** 被删除条目对应的高亮消失
- **AND** 剩余条目卡片重新按对应原文位置和防重叠规则布局

### Requirement: 手动病例提交后条目锁定
系统 MUST 在当前病例提交成功后阻止操作员继续新增、编辑或删除该病例的质控条目卡片。

#### Scenario: 提交成功后锁定当前病例条目
- **GIVEN** 操作员确认提交当前手动病例
- **WHEN** 手动标注提交成功
- **THEN** 当前病例不再允许新增、编辑或删除质控条目卡片
- **AND** 页面推进到下一条任务或展示计划完成状态

### Requirement: 手动提交确认交互
系统 MUST 在手动工作台提交前统一展示二次确认弹窗，并 SHALL 根据当前标注条目数量展示不同确认文案。提交确认弹窗 SHALL 使用系统反馈组件的 Dialog 样式。

#### Scenario: 有问题条目时确认提交
- **GIVEN** 操作员正在查看手动任务
- **AND** 当前已有 N 条标注条目且 N 大于 0
- **WHEN** 操作员点击“提交”
- **THEN** 页面展示确认文案 `当前病历已标记{N}条质控问题，是否确认提交？提交后将不能新增/修改质控建议`

#### Scenario: 无问题条目时确认提交
- **GIVEN** 操作员正在查看手动任务
- **AND** 当前标注条目数为 0
- **WHEN** 操作员点击“提交”
- **THEN** 页面展示确认文案 `当前病历未标记质控问题，是否确认提交？提交后将不能新增/修改质控建议`

#### Scenario: 取消二次确认
- **GIVEN** 手动提交确认弹窗已经展示
- **WHEN** 操作员取消确认
- **THEN** 页面停留在当前任务
- **AND** 不发送手动标注提交请求

#### Scenario: 确认二次提交
- **GIVEN** 手动提交确认弹窗已经展示
- **WHEN** 操作员确认提交
- **THEN** 页面发送手动标注提交请求
- **AND** 提交按钮在请求期间展示加载态或阻止重复点击

### Requirement: 手动模式质控规则上下文
系统 SHALL 在手动工作台通过左侧病历原文 title 区的“查看质控规则”入口展示任务 API 返回的相关质控规则上下文，并 MUST NOT 允许操作员编辑质控规则。

#### Scenario: 手动任务包含质控规则
- **GIVEN** 手动任务 payload 包含一个或多个质控规则
- **WHEN** 操作员点击“查看质控规则”
- **THEN** 页面在按钮右侧弹出 popover
- **AND** popover 以只读方式展示规则分类、规则内容和分值

#### Scenario: 手动任务没有质控规则
- **GIVEN** 手动任务 payload 中质控规则列表为空
- **WHEN** 操作员点击“查看质控规则”
- **THEN** popover 展示低干扰的无规则状态

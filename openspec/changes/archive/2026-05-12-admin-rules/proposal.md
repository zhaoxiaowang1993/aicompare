## Why

管理员当前缺少集中维护质控规则的入口，导致标注详情页展示依据无法由业务人员持续更新。根据 `openspec/specs/SPEC.md` 3.7，本次变更补齐管理员质控规则管理模块，使全部计划统一展示当前生效规则，并为后续质控规则迭代提供基础数据。

## What Changes

- 新增管理员规则管理页 `/admin/rules`，用于查看、搜索、分页、新增、编辑和删除质控规则。
- 新增质控规则字段与交互：规则分类、规则内容、分值。
- 规则分类为单选枚举，包含入院病历、首次病程记录、上级医师查房记录、日常病程、出院记录。
- 规则内容为文本，分值为文本。
- 支持批量导入质控规则，并提供 CSV 模板下载。
- 标注详情页读取并展示当前最新生效质控规则，作为操作员对比 A/B 质控结果时的参考依据。
- 新增后端管理员规则管理 API，覆盖列表查询、创建、更新、删除、CSV 模板下载和批量导入等管理操作。
- 新增数据库持久化结构保存质控规则，支持后续统一查询当前生效规则。
- 不引入规则启用或停用逻辑。
- 不引入 breaking changes。

## Project Scope

### In Scope

- 管理员可管理全局质控规则列表，包括创建、读取、更新和删除。
- 规则列表支持按名称模糊搜索和分页展示。
- 管理员可下载 CSV 模板并批量导入质控规则。
- 操作员标注详情页展示最新生效规则。
- 前端先以 mock 假数据完成页面与交互验收，再接入真实后端 API。
- 设计阶段产出 Pencil 设计文件规划，并在前端实现阶段进行 1:1 还原审计。

### Out of Scope

- 本期不做规则版本冻结，历史计划和历史标注不绑定旧版本规则。
- 本期不做规则审批流、发布流、灰度生效或多租户规则隔离。
- 本期不做规则启用、停用或生效状态管理。
- 本期不做规则导出、批量编辑或审计日志。
- 本期不做规则表达式引擎或自动评分执行，仅维护展示依据。
- 本期不修改标注结论、统计报表或计划状态流转的既有业务规则。

## Capabilities

### New Capabilities

- `admin-rules-management`: 管理员维护全局质控规则，支持单条增删改查和 CSV 批量导入，并向标注详情页提供最新规则展示依据。

### Modified Capabilities

- None.

## Impact

- 前端路由与页面：新增 `/admin/rules` 规则管理页，并更新操作员标注详情页的数据流以加载最新规则。
- 前端模块：新增规则管理页面与相关组件，新增 `frontend/src/types/rules.ts` 类型定义，并在 `frontend/src/lib/api.ts` 中补充规则管理 API 封装。
- 后端路由：新增 `backend/app/routers/rules.py`，并在 FastAPI 应用中注册规则管理路由。
- 后端数据层：新增规则相关 schema、model、service，并新增或调整数据库初始化逻辑以支持 `quality_rules` 表。
- API 范围：新增管理员专用 `/admin/rules` 接口，覆盖列表、详情、创建、更新、删除、CSV 模板下载和批量导入；新增标注详情页读取最新规则所需的查询路径。
- 设计产物：补充管理员规则管理页、CSV 导入流程、标注详情页规则展示状态的 Pencil 设计规划。
- 测试与验收：覆盖前端 mock 流程验收、设计稿 1:1 还原审计、后端 API 测试、CSV 导入校验和前后端联调验证。
